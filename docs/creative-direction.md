## 1. Core Read

Nick Huo / HuoHuoOvO should not be introduced as a portfolio category. The strongest read is a person moving through several fields at once:

- games and player communities
- official content and private language
- AI tools and human judgment
- visual experiments and practical systems
- memories, emotions, and unfinished thoughts

The previous `Signal Workbench` concept was accurate but too narrow as a whole-site frame. It describes a work method: how Nick reads community noise, turns it into judgment, and builds systems around it. It should live inside Work, not define the entire person.

The new homepage should feel like a personal field, not a single dashboard. It should show that Work is important, but not enough. Ideas, Art, Thinking, Emotion, and Tools are not supporting decorations; they are equal parts of the field.

## 2. Three Keywords

### Field

The site is a field, not a funnel. It should hold multiple dimensions without forcing them into one career narrative. A field can contain work, images, systems, notes, fragments, and emotional residue.

### Movement

The identity comes from moving between things: DOTA2 and AI agents, official publishing and private text, visual experiments and operational systems, practical tools and things that cannot be fully measured.

### Residue

Residue keeps the page human. Not everything on the site needs to prove utility. Some things remain because they once mattered: DOTA2 memory, unfinished text, low-frequency emotion, future-reunion imagination, and fragments that do not become projects.

## 3. Recommended Concept

### HuoHuo Field / 火火场域

The whole-site concept should be:

> **HuoHuo Field**
> A personal field of work, ideas, art, thoughts, emotions, and tools.

In Chinese:

> **火火场域**
> 一个整理工作、理念、艺术、思考、情绪与工具的个人场域。

This concept is broader than a portfolio and warmer than a dashboard. It lets the homepage answer:

1. Who is Nick Huo / HuoHuoOvO?
2. What does he work on?
3. What does he care about?
4. Where do DOTA2, Boommunity, player signals, AI agents, visual experiments, writing, and tools belong?
5. What remains outside projects, metrics, and resumes?

`Signal Workbench` remains important, but it becomes a Work-zone concept.

## 4. Why This Concept Fits

HuoHuo Field fits because it does not collapse Nick into one professional label.

Work:
Boommunity, DOTA2 community operations, player signals, creator ecosystems, content publishing, internal workflows, and AI Agent systems all sit here. This is where signals become systems.

Ideas:
Nick has specific judgments about community operations, AI tools, future interfaces, player trust, and product systems. These should appear as ideas, not only as project outcomes.

Art:
Visual experiments, ComfyUI, DOTA2 visual studies, editorial taste, print texture, illustration, poster thinking, and color research need a place that is not merely "assets".

Thinking:
Longer notes about AI, DOTA2, work, love for games, future interfaces, and human-machine collaboration belong in a field-notes layer, not a generic blog.

Emotion:
Some material is not explanatory. It is memory, loss, time, romance, and fragments. This layer prevents the site from becoming a work dashboard.

Tools:
Hermes, Codex, Obsidian, ComfyUI, Figma, Trae, Next.js, local agents, automation, and Workdesk are instruments for thinking, making, organizing, and preserving memory. They are not badges.

## 5. Visual Language

### Color Tendency

Use a field with multiple surfaces:

- Work surface: deep ink / near black.
- Archive surface: warm paper / bone.
- Signal accent: burnt red or cherry red.
- Tool signal: dusty cyan or pale turquoise.
- Memory tone: muted amber or olive.

The page should avoid a one-note palette. Dark operational sections and warmer archive/emotion sections should coexist.

### Typography Tendency

Typography should be the main visual engine:

- Strong display treatment for `Nick Huo` and `HuoHuoOvO`.
- Chinese text should remain clear, grounded, and warm.
- Monospace should be limited to timestamps, coordinates, status, tool names, and small labels.
- Avoid generic sci-fi display fonts.

### Graphic System

The graphic language should combine:

- field map
- index
- coordinates
- thin signal lines
- status strips
- fragment IDs
- archive labels
- restrained game UI order

Do not use generic AI blobs, glowing orbs, fake terminal typing, 3D cubes, purple-blue SaaS gradients, or fake draggable OS windows.

### Page Mood

The mood should be:

- personal field
- archive system
- workbench
- visual lab
- light game UI order

It should be visually attractive, but the beauty should come from hierarchy, contrast, rhythm, texture, and meaning, not from decorative effects.

## 6. Information Architecture

The homepage should use six top-level dimensions.

### 1. Work / 工作

Reality-facing projects, career work, community operations, content work, and systems.

Includes:

- Boommunity
- DOTA2 community and content operations
- player signals
- creator / KOL ecosystem
- game community judgment
- AI Agent workflow
- official content and event publishing
- internal tools and workflows

Core line:

> Work is where signals become systems.
> 工作是信号变成系统的地方。

`Signal Workbench` belongs here:

> A work method for reading community noise, turning it into judgment, and building systems around it.

### 2. Ideas / 理念

Short judgments, beliefs, and methods about games, AI, tools, communities, and future interfaces.

Examples:

- Community operation is not posting; it is translating player signals.
- Future apps will organize around user intent, not only pages.
- AI tools matter when they reveal judgment earlier.
- Game communities are built on trust, not just reach.

### 3. Art / 艺术

Visual experiments and aesthetic expression.

Includes:

- ComfyUI
- image generation
- DOTA2 visual experiments
- posters / illustration / type / color studies
- aesthetic archive
- editorial illustration, retro-future, paper texture, restrained visual studies

### 4. Thinking / 思考

More complete writing, observations, and methods.

This is not a normal blog. It is closer to field notes.

Possible labels:

- Field Notes
- Thinking Notes
- Notes
- Observations

### 5. Emotion / 情绪

The private-adjacent, abstract, unresolved layer.

Thinking is "how I think."
Emotion is "what remains on me."

Includes:

- DOTA2 memory
- time
- loss
- future reunion
- unfinished fragments
- low-frequency text
- things that should not be over-explained

### 6. Tools / 工具

Instruments for thinking, making, organizing, executing, and preserving memory.

Includes:

- Hermes
- Codex
- Claude Code
- Obsidian
- ComfyUI
- Figma
- Trae
- Next.js
- local agents
- automation
- Workdesk

Tools are instruments, not badges.

## 7. Homepage V1 Structure

### 1. Hero / 首屏

Show the person before the projects.

Required content:

- `Nick Huo`
- `HuoHuoOvO / 火火`
- a core expression
- a multidimensional field map or coordinate system

Recommended core expression:

> I move between games, communities, tools, images and unfinished thoughts.

Chinese:

> 我在游戏、社区、工具、图像和未完成的想法之间移动。

### 2. Dimension Map / 多维地图

Show six dimensions:

- Work / 工作
- Ideas / 理念
- Art / 艺术
- Thinking / 思考
- Emotion / 情绪
- Tools / 工具

This should not be a normal bento grid. It should feel like a coordinate map, archive index, or field layer system.

### 3. Featured Work / 工作分支

Show Work as one important branch, not the whole site.

Include:

- Boommunity
- DOTA2 community / content operations
- player signal / community feedback loop
- Agent workflow

This is where `Signal Workbench` visual language can appear.

### 4. Ideas / Thinking Preview

Show judgments as short sentence indexes.

This section should prove that Nick has his own positions, not just execution experience.

### 5. Art / Visual Lab Preview

Show the visual layer without depending on missing image assets.

If real public images are unavailable, use no-image structures, CSS textures, typographic fragments, and optional covers.

### 6. Emotion / Residue Section

Keep this light.

Example tone:

> Not everything I keep is useful.
> Some things stay because they once meant something.

Chinese:

> 不是所有留下来的东西都有用。
> 有些东西留下来，只是因为它曾经重要过。

### 7. Archive Teaser

Archive should eventually hold work, ideas, art, thinking, emotion, and tools.

For v1, keep `/works` and existing content. Do not force a full migration.

### 8. Handoff

Contact, but in site language.

It can include:

- open to game community / AI workflow / visual system / tool-building conversations
- GitHub / email / social links when available
- a short working-with-me line

## 8. Interaction / Motion Direction

Motion should support field navigation, not spectacle.

Recommended:

- one calm load-in for the hero field
- hover/focus on dimension nodes reveals emphasis
- subtle line or status changes
- no scroll-jacking
- no fake terminal typing
- no draggable windows
- no heavy 3D
- no animation library for v1

Reduced-motion mode must leave all content visible and complete.

Suggested future dimensions:

```ts

  | "work"

  | "public"
```

Potential fields:

- title
- slug
- dimension
- status
- year
- oneLine
- description
- tools
- relatedWorlds
- publicBoundary
- featured
- order

For homepage v1, static arrays are enough. Do not migrate the whole MDX system yet.

## 10. Public Boundary

Do not publish private Obsidian memory directly.

Do not expose:

- health
- family
- finance
- private emotional details
- spouse information
- overly specific personal life details
- unconfirmed company-internal information

Usable public material:

- public career experience
- public or carefully rewritten project material
- personal methods
- visual experiments
- writing directions
- tool systems
- public-safe DOTA2 / community / AI / personal-site content

Future content items should include `publicBoundary`.

## 11. Things to Avoid

- Generic designer portfolio.
- Developer portfolio with project cards and GitHub badges.
- AI-tool influencer landing page.
- DOTA2 fan page.
- Fake OS desktop.
- Fake terminal typing.
- Full dashboard product.
- Generic cyberpunk.
- Purple-blue SaaS gradients.
- Glowing AI blobs.
- Heavy 3D.
- Bento-card sprawl.
- Turning every text into career reflection.
- Overusing "low-temperature burn" as the master metaphor.
- Claiming planned or partial projects are mature.
- Publishing sensitive Obsidian context.

## 12. Implementation Notes

Homepage v1 can be implemented without over-refactoring.

Keep:

- Next.js 16
- Tailwind v4
- MDX
- `/works`
- `content/works`
- old docs
- existing project structure where possible

Do not:

- install animation libraries
- add complex state management
- migrate the whole content system
- delete old docs
- implement fake OS
- add complex 3D
- use scroll-jacking

Allowed:

- update `src/app/page.tsx`
- add necessary homepage components
- add homepage static data
- add small CSS variables / Tailwind tokens
- add simple SVG / CSS visual anchors
- make existing work covers optional
- add `docs/homepage-v1-plan.md` if useful

Suggested components:

- `HeroField`
- `DimensionMap`
- `DimensionNode`
- `FeaturedWork`
- `SignalWorkbenchPreview`
- `IdeasPreview`
- `VisualLabPreview`
- `ResidueSection`
- `ArchiveTeaser`
- `Handoff`

Do not over-abstract before the v1 shape is proven.

## 13. Success Standard

Homepage v1 succeeds if a visitor understands:

> Nick Huo / HuoHuoOvO is a person moving between games, communities, AI tools, visual experiments, personal writing, and emotional memory.

It should also be clear that:

- Work is important.
- Boommunity is a major Work project.
- player signal is a Work method.
- DOTA2 belongs to Work, memory, and visual history.
- Art / Thinking / Emotion / Tools are not side decorations.
- The site is not a resume, not a project-card collection, and not an AI tool showcase.

If the visitor only remembers "player signal workbench", the concept failed.
