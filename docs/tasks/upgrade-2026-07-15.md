## Task 1: 真实加载字体（当前最大缺陷）

现状：`globals.css` 里 `--font-space-grotesk: "Space Grotesk", ...`、`--font-ibm-plex-mono: "IBM Plex Mono", ...` 只是声明，字体从未被加载，线上实际渲染 Segoe UI/Arial 回退。

要求：
- 在 `src/app/layout.tsx` 用 `next/font/google` 加载 `Space_Grotesk`（weights: 400, 500, 700）和 `IBM_Plex_Mono`（weights: 400, 500），`display: "swap"`，通过 `variable` 挂 CSS 变量到 `<html>`。
- `Noto Sans SC` 也用 next/font 加载（subsets 用 `["latin"]` 即可——CJK 字体 Google 会全量提供 glyph，不要和 TypeScript subset 类型较劲；如报 subset 错误，加 `preload: false`）。中文字重只要 400 和 700，控制体积。
- 更新 `globals.css` 中三个 `--font-*` 变量，让它们引用 next/font 生成的变量（如 `var(--font-space-grotesk-next)`），保留现有系统字体回退链。
- next/font 在 build 时下载并自托管，静态导出后不依赖 Google CDN（部署目标用户在中国，不能有 fonts.googleapis.com 运行时请求）。构建后检查 `out/` 中 HTML，确认没有指向 googleapis 的 link 标签，字体文件在 `_next/static` 下。
